Implantação de Recursos no Azure usando modelos JSON do Resource Manager

O Azure Resource Manager (ARM) é a plataforma de implantação e gerenciamento de recursos do Azure. Ele permite que você crie, atualize e exclua recursos em seu ambiente do Azure usando modelos JSON. Esses modelos são arquivos de texto que definem a infraestrutura e a configuração dos recursos que você deseja implantar.

1. Autenticação no Azure

Primeiro, entre no Azure usando a CLI ou o PowerShell.

CLI do Azure:
az login

PowerShell do Azure:
Connect-AzAccount

2. Criação do Grupo de Recursos

Em seguida, defina o grupo de recursos onde os recursos serão implantados. O grupo de recursos é um contêiner lógico que agrupa os recursos relacionados para facilitar o gerenciamento. Você pode criar um grupo de recursos usando o seguinte comando:

CLI do Azure:
az group create \
  --name {nome_do_grupo} \
  --location "{localização}"

PowerShell do Azure:
New-AzResourceGroup `
  -Name {name of your resource group} `
  -Location "{location}"

3. Lógica de Implantação de Modelos

Para iniciar uma implantação de modelo no grupo de recursos, use o comando da CLI do Azure az deployment group create ou do Azure PowerShell New-AzResourceGroupDeployment.

Para rodar os dois comandos, você sempre precisará informar três itens básicos:

  • O grupo de recursos;
  • A região;
  • Um nome para a implantação (isso serve para você conseguir achar o que fez no histórico depois).

Além disso, para facilitar a sua vida, você pode criar um "atalho" (uma variável) que guarda o endereço do arquivo no seu sistema. Desse jeito, você não precisa ficar digitando aquele caminho longo toda vez que for executar o comando de instalação. Veja o exemplo abaixo:

templateFile="{provide-the-path-to-the-template-file}"

az deployment group create \
  --name blanktemplate \
  --resource-group myResourceGroup \
  --template-file $templateFile

Laboratório: Evolução da Implantação

Após entender os conceitos básicos acima, siga os passos abaixo para evoluir sua infraestrutura como código.

1. Configuração Inicial do Ambiente

Para evitar a repetição do escopo em cada comando, defina o grupo de recursos padrão da sessão:

az configure --defaults group="departamentoTI"

2. Histórico de Implantações (Deployments)

Dica de Sintaxe Bash: Para manter comandos longos do Azure CLI organizados e fáceis de ler, sem quebrar a sintaxe, a melhor prática é usar a barra invertida. No Bash (como no MINGW64), a barra invertida atua como um caractere de escape para a quebra de linha. Ela informa ao interpretador do terminal que a instrução atual ainda não terminou e deve ser lida em conjunto com a próxima linha. Se colocar o código sem a barra invertida pode dar erro no interpretador no terminal.

Implantação Base (Blank Template)

templateFile="azuredeploy.json"
today=$(date +"%d-%b-%Y")
DeploymentName="blanktemplate-"$today

az deployment group create \
  --name $DeploymentName \
  --template-file $templateFile \
  --resource-group "departamentoTI"

Implantação: Adicionando Storage

templateFile="azuredeploy.json"
today=$(date +"%d-%b-%Y")
DeploymentName="addstorage-"$today

az deployment group create \
  --name $DeploymentName \

3. Arquivo de Modelo (azuredeploy.json)

Abaixo está o modelo JSON final utilizado nas implantações acima, contemplando as funções, variáveis, recursos e a seção de outputs:

{
    "$schema": "http://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
    "contentVersion": "1.0.0.0",
    "parameters": {
        "storageName": {
            "type": "string",
            "minLength": 3,
            "maxLength": 24,
            "metadata": {
                "description": "The name of azure storage resource"
            }
        },
        "storageSKU": {
            "type": "string",
            "defaultValue": "Standard_LRS",
            "allowedValues": [
                "Standard_LRS",
                "Standard_GRS",
                "Standard_RAGRS",
                "Standard_ZRS",
                "Premium_LRS",
                "Premium_ZRS",
                "Standard_GZRS",
                "Standard_RAGZRS"
            ]
        }
    },
    "functions": [],
    "variables": {},
    "resources": [
        {
            "type": "Microsoft.Storage/storageAccounts",
            "apiVersion": "2021-09-01",
            "name": "[parameters('storageName')]",
            "tags": {
                "displayName": "Test StorageAccount for learning azure"
            },
            "location": "southafricanorth",
            "kind": "StorageV2",
            "sku": {
                "name": "[parameters('storageSKU')]"
            }
        }
    ],
    "outputs": {
        "storageEndpoint": {
            "type": "object",
            "value": "[reference(parameters('storageName')).primaryEndpoints]"  
        }
    }
}
